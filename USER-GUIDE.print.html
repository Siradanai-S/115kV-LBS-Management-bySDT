📋 Sales Department (ฝ่ายขาย)

หน้าที่: เก็บข้อมูลลูกค้า → รวบรวมเป็นใบขอ (SR) ส่งให้ฝ่ายโครงการ → ติดตาม PO/สัญญาจากลูกค้า แยกรายลูกค้า

หน้าหลัก (Sales Department) — แสดงเฉพาะลูกค้าที่ยังไม่ได้รวบรวม

  1. กด + เพิ่มลูกค้า กรอก: ชื่อ, ประเภทลูกค้า, ประเภทงาน, ผู้ติดต่อ, จำนวน LBS ที่ต้องการ, วันที่ต้องการ, Term of payment, Scope, ชื่อโครงการ และ สถานที่ติดตั้ง
  2. สถานที่ติดตั้ง — เพิ่มได้หลายแห่ง: กด “+ เพิ่มสถานที่” เพื่อใส่หลายไซต์ในลูกค้าเดียว
  3. ลูกค้าถูกจัดกลุ่มตามประเภท (ราชการ / เอกชน / รัฐวิสาหกิจ) · การ์ดลูกค้าแสดง 📁 ชื่อโครงการใต้ชื่อ
  4. ติ๊กเลือกลูกค้า (เลือกได้หลายราย) → กด รวบรวมส่งฝ่ายโครงการ → ระบบออกเลข SR-YYYY-NNN

แท็บกิ่งย่อย → Sales Requisition & ติดตาม PO

  • กล่อง ⏳ รอฝ่ายโครงการสร้าง Stock และ ✓ สร้าง Project Stock แล้ว — พับได้ เริ่มต้นซ่อน · ในกล่อง แต่ละ SR No. พับได้ (เริ่มต้นซ่อน)
  • กางใบ SR → ตารางลูกค้าในใบ (เส้นกริดสีเทา) — กรอก เลขที่ PO ลูกค้า + สถานะสัญญา ได้ แยกรายลูกค้า
  • ↩️ กล่อง “ฝ่ายโครงการตีกลับให้แก้”: ถ้าฝ่ายโครงการส่ง SR กลับมา จะขึ้นที่นี่พร้อมเหตุผล → แก้ข้อมูลแล้วกด “✓ แก้แล้ว — ส่งกลับฝ่ายโครงการ” เพื่อส่ง SR กลับเข้า Inbox ฝ่ายโครงการ

🔗 Flow: กด “รวบรวมส่ง” → SR ไปโผล่ใน Inbox ของ Project ทันที (มีแจ้งเตือน) · ลูกค้าถูกย้ายออกจากหน้าหลัก Sales

📐 Project Department (ฝ่ายโครงการ)

หน้าที่: รับ SR → สร้าง Project Stock → ทำ BOM (หลายครั้ง) → ส่งจัดซื้อ → ส่งงานต่อตาม Flow → คลัง + เบิกส่งมอบ Service

หน้าหลัก (Project Department) — Inbox ของ SR ที่ฝ่ายขายส่งมา (ตารางแยกตาม SR No.)

  1. แต่ละ SR No. เป็นบล็อก พับได้ (เริ่มต้นซ่อน) — กางเห็นตารางลูกค้า (ชื่อ/ประเภท/งาน/LBS/ชื่อโครงการ/วันที่/สถานที่/ผู้ติดต่อ/Scope)
  2. ติ๊กเลือก SR (เลือกได้หลายใบ) → กด สร้าง Project Stock
  3. Popup: ตั้งชื่อโครงการ + กรอกจำนวน LBS เป้าหมาย (ระบบเติมยอดรวมลูกค้าให้ แก้ได้) + แก้ Stock No./Job No./BOM No.
  4. ↩️ ส่งกลับฝ่ายขาย: ถ้าข้อมูล SR ไม่ถูกต้อง กด “↩️ ส่งกลับฝ่ายขาย” ที่หัวใบ → ระบุเหตุผล → SR ออกจาก Inbox ไปให้ฝ่ายขายแก้

แท็บกิ่งย่อย → Stock No. (การ์ดรายโครงการ พับได้ เริ่มต้นซ่อน) — กางการ์ดเห็นตามลำดับ Flow:

  • สถานะ + ปุ่มส่งต่อ (บนสุด): แถบเฟส + ปุ่มเดียวจบ “เสร็จ & ส่งต่อ → [ฝ่ายถัดไป]” (ปิดงานที่เหลือให้อัตโนมัติ + ตรวจเงื่อนไข + ส่งต่อในคลิกเดียว) · ← ตีกลับ (ระบุเหตุผล) · รับงาน (เคลียร์ badge งานเข้าใหม่)
  • Project Budget [Stock No.]: งบประมาณรวม + มูลค่าวัสดุ BOM + งบประมาณคงเหลือ (ตัดอัตโนมัติจากมูลค่า BOM · ติดลบ=แดง)
  • ตรวจสอบจำนวน LBS (การ์ดกระชับ): ยอด LBS ใน BOM ต้องตรง เป้าหมาย Stock (เขียว=ตรง / แดง=ขาด-เกิน)
  • Material List (BOM) — หลายครั้ง: ปุ่ม “เพิ่ม Material List (BOM) ครั้งใหม่” · การ์ดแต่ละครั้งชื่อ BOM-[Stock No.] (ครั้งN) พับได้ เริ่มต้นซ่อน + ปุ่ม “ส่ง BOM (ครั้งN) ให้จัดซื้อ” แยกกัน (ส่งแล้วล็อกรอบนั้น)
  • รายการ BOM: Epicor Code, Description, หมวด (LBS/Accessory), Due Date, IUM, จำนวน, สกุลเงิน THB/USD/EUR → เลือก USD/EUR ระบบ ดึงอัตราแลกเปลี่ยนปัจจุบันอัตโนมัติ (กรอก/กดดึงซ้ำ 🔄 ได้) → Total คิดเป็นบาท · คอลัมน์ “สถานะ PO (แจ้งกลับ Project)” โชว์สถานะ + เลข PO จากจัดซื้อ
  • Serial LVB / Serial OM: ช่องกรอกปรากฏ เฉพาะหมวด = LBS
  • มี 📜 Audit Timeline ประวัติส่งงาน/ตีกลับ/รับงาน

แท็บกิ่งย่อย → คลังสินค้า (Inventory) — คลังแยกตาม Project Stock (การ์ดพับได้ เริ่มต้นซ่อน)

  1. รับเข้าอัตโนมัติ: เมื่อฝ่ายจัดซื้ออัปเดต PO เป็น Delivered ของจะเข้าคลังโครงการให้เอง
  2. 📤 เบิก (อ้าง SR-No.): Popup เลือก SR → เลือกลูกค้าได้หลายเจ้า → กรอกจำนวนเบิกแยกต่อลูกค้า (ไม่เกินคงเหลือ) → บันทึก = ตัดสต็อก + สร้าง ใบเบิก 1 ใบต่อลูกค้า เข้าคิว “รอส่งมอบ Service”

🔗 Flow (ส่งต่อเฟส project → purchasing): งานครบ + Job No. + มี BOM + ส่ง BOM ครบทุกครั้ง + LBS ตรงเป้า → กด “เสร็จ & ส่งต่อ” บนการ์ด · ฝ่ายจัดซื้อเห็นเฉพาะรอบ BOM ที่ส่งแล้ว

🛒 Purchasing Department (ฝ่ายจัดซื้อ)

หน้าที่: รับ BOM (รอบที่ส่งแล้ว) → ออก PO → อัปเดตสถานะของเข้า แจ้งกลับฝ่ายโครงการ

หน้าหลัก — การ์ดต่อ Stock/BOM (พับได้, เริ่มต้นซ่อน)

  1. กางการ์ด → เห็น เฉพาะรายการ BOM รอบที่ส่งมาแล้ว (มีป้าย “(ครั้งN)”)
  2. กด ออก PO → เลือกรายการ + ระบุจำนวนต่อรายการ (แยกใบ/บางส่วนได้) → ต้องกรอกเลขที่ PO เอง (จำเป็น) จึงบันทึกได้ + แนบไฟล์ PDF → มูลค่ารวมคิดอัตโนมัติ
  3. อัปเดต สถานะ PO ราย BOM (Awaiting → PO Received → Delivered) — ฝ่ายโครงการเห็นเลข PO + เปิดไฟล์ได้

แท็บกิ่งย่อย → BOM Delivered completed — กลุ่มตาม Stock No. พับได้ เริ่มต้นซ่อน (ปุ่มขยาย/ย่อทั้งหมด)

🔗 Flow (purchasing → service): เมื่อ ทุกรายการ BOM = Delivered → กด “เสร็จ & ส่งต่อ” ไป Service · ตีกลับฝ่ายโครงการได้ (ระบุเหตุผล)

🛠️ Service Department (ฝ่ายบริการ)

หน้าที่: รับงานตามใบเบิก → จัดทีม + เวลาจริง → ทำ Report ส่งมอบ + Check-in · หน้าจอแบ่งเป็น 3 หัวข้อ (พับได้) ตามสถานะงาน

  1. 📥 Pending Actions — รอกดรับงาน: ใบเบิกที่ฝ่ายโครงการส่งมา → ดูรายการ + ปริ้น ใบเบิก / D/O / Handover ไปตรวจรับที่คลัง → กด ✓ รับงาน
  2. 👷 Assigned Teams — จัดทีม + เวลาจริง: เลือกทีม 1–2 คน (คนติดคิวขึ้น ⚠) + กรอก actual Start/End + ติ๊ก เช็กลิสต์หน้างาน (ตรวจรับวัสดุ/ติดตั้ง/Commissioning) → บันทึก (ลงคิวปฏิทินอัตโนมัติ)
  3. ✅ Completed Projects — ทำ Report + Check-in: กรอกผู้รับมอบ + วันที่ + 📍 Check-in Location (กด 📡 ดึง GPS หรือกรอก lat/lng เอง — จำเป็นต้องมีพิกัดก่อนจึงทำ Report ได้) + แนบไฟล์ → กด ✓ ทำ Report → Completed (ลงนาม DO) · การ์ดที่เสร็จมีลิงก์เปิด Google Maps
  • ↩️ คืนของเข้าคลัง: ถ้าเบิกเกิน/ไม่ได้ติดตั้ง กดปุ่ม “คืนของเข้าคลัง” ที่การ์ด → กรอกจำนวนคืน → ของกลับเข้าคลังโครงการ
  • 🏁 ปิดงานโครงการ (Closed): ปุ่มขึ้นเมื่อโครงการอยู่เฟส service และ ส่งมอบครบทุกใบเบิก → กดปิดงาน (ถ้ายังเหลือบางใบจะปิดไม่ได้)
  • 🛡️ ใบรับประกัน: ออกเลขรับประกัน + วันสิ้นสุด + พิมพ์ใบรับประกันได้หลังส่งมอบ
  • ทีมงาน & Scheduling Calendar: เพิ่ม/ลบสมาชิก (รหัสพนักงาน + ตำแหน่ง + โทร) · ปฏิทินรายเดือน/Roster

🔗 Flow (service → closed): ปิดงานได้เมื่อ ทุกใบเบิกส่งมอบครบ (delivered) + ลงนาม DO — กันปิดงานก่อนกำหนดเมื่อมีหลายลูกค้า/หลายใบเบิก · งานฝ่ายบริการ tick ให้อัตโนมัติตามความคืบหน้าของทุกใบเบิก

🗺️ Map Tracking (แผนที่ติดตามงานเสร็จ)

เมนูย่อยใต้ Service Department → แผนที่ประเทศไทย (Leaflet + OpenStreetMap) ปักหมุด Completed Project ตามพิกัด Check-in ที่ทีมบันทึกตอนทำ Report · คลิกหมุดดูข้อมูล (Stock No. / ลูกค้า / วันที่ / พิกัด)

ต้องมีอินเทอร์เน็ต (โหลดแผนที่) · หมุดจะขึ้นเฉพาะงานที่ทีมกด Check-in ตอนทำ Report

🔗 สรุป Flow เชื่อมโยงทั้งระบบ

ลูกค้า ──(Sales รวบรวม)──▶ Sales Requisition (SR)  [ติดตาม PO/สัญญา รายลูกค้า · ตีกลับได้]
                                  │ ส่งให้โครงการ   ◀── ↩️ ตีกลับ SR ให้แก้
                                  ▼
   (Project เลือกหลาย SR + ถาม LBS) ──▶ Project Stock + BOM (หลายครั้ง · THB/USD/EUR)
              │ ส่ง BOM (รายครั้ง)                  │ เบิก(อ้าง SR) แยกต่อลูกค้า = 1 ใบเบิก/ลูกค้า
              ▼                                      ▼  (ตัดสต็อกโครงการ)
        Purchasing (ออก PO · เลข PO เอง)   Service (3 หัวข้อ: Pending → Assigned → Completed+Check-in)
              │ Delivered ครบ                       │ Report + Check-in (GPS) · คืนของได้
              ▼                                      ▼
   คลัง: รับตาม PO อัตโนมัติ ──────────────▶  ปิดงาน (Closed) เมื่อส่งมอบครบทุกใบเบิก ──▶ 🗺️ Map Tracking

เฟสของ Project Stock (Workflow): project → purchasing → service → closed — ปุ่มส่งต่ออยู่บนการ์ด Stock (กดทีเดียว ปิดงานที่เหลือ + ส่งต่อ)

เงื่อนไขส่งงานต่อ (Gate) แต่ละเฟส:

จาก → ไปต้องผ่านเงื่อนไข
project → purchasingงานฝ่ายโครงการครบ + มี Job No. + มี BOM + ส่ง BOM ครบทุกครั้ง + LBS ใน BOM = เป้าหมาย
purchasing → serviceงานฝ่ายจัดซื้อครบ + ทุกรายการ BOM = Delivered
service → closedลงนาม DO + ส่งมอบครบทุกใบเบิก (delivered ทุกใบ)

ส่ง-รับ-ตอบกลับ ข้ามฝ่าย:

ส่งงานต่อ (Forward): ปุ่ม “เสร็จ & ส่งต่อ” บนการ์ด Stock (ผ่าน Gate เท่านั้น) → ปลายทางเห็น “งานเข้าใหม่”

รับงาน (Ack): ฝ่ายปลายทางกด “รับงาน” เคลียร์ badge

ตีกลับ (Reject): ส่งกลับเฟสก่อนหน้า (ระบุเหตุผล) · Sales↔Project มีช่องตีกลับ SR · Service มี “คืนของเข้าคลัง”
