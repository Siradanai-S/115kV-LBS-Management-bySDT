📋 Sales Department (ฝ่ายขาย)

หน้าที่: เก็บข้อมูลลูกค้า → รวบรวมเป็นใบขอ (SR) ส่งให้ฝ่ายโครงการ → ติดตาม PO/สัญญาจากลูกค้า แยกรายลูกค้า

หน้าหลัก (Sales Department) — แสดงเฉพาะลูกค้าที่ยังไม่ได้รวบรวม

  1. กด + เพิ่มลูกค้า กรอก: ชื่อ, ประเภทลูกค้า, ประเภทงาน, ผู้ติดต่อ, จำนวน LBS ที่ต้องการ, วันที่ต้องการ, Term of payment (เงื่อนไขชำระเงิน), Scope
  2. สถานที่ติดตั้ง — เพิ่มได้หลายแห่ง: กด “+ เพิ่มสถานที่” เพื่อใส่หลายไซต์ในลูกค้าเดียว
  3. ลูกค้าถูกจัดกลุ่มตามประเภท (ราชการ / เอกชน / รัฐวิสาหกิจ)
  4. ติ๊กเลือกลูกค้า (เลือกได้หลายราย) → กด รวบรวมส่งฝ่ายโครงการ → ระบบออกเลข SR-YYYY-NNN

แท็บกิ่งย่อย → Sales Requisition & ติดตาม PO

  • กล่อง ⏳ รอฝ่ายโครงการสร้าง Stock / ✓ สร้าง Project Stock แล้ว
  • แต่ละ SR แสดง ตารางลูกค้าในใบ (เส้นกริดสีเทา) — กรอก เลขที่ PO ลูกค้า + สถานะสัญญา ได้ แยกรายลูกค้า (1 ใบมีหลายลูกค้า สถานะต่างกันได้)

🔗 Flow: กด “รวบรวมส่ง” → SR ไปโผล่ใน Inbox ของ Project ทันที · ลูกค้าถูกย้ายออกจากหน้าหลัก Sales (ไปอยู่ใต้ SR)

📐 Project Department (ฝ่ายโครงการ)

หน้าที่: รับ SR → สร้าง Project Stock → ทำ BOM (ได้หลายครั้ง) → ส่งให้จัดซื้อ → ออกใบเบิกติดตั้ง + ส่งมอบให้บริการ

หน้าหลัก (Project Department) — Inbox ของ SR ที่ฝ่ายขายส่งมา

  1. ติ๊กเลือก SR (เลือกได้หลายใบ) → กด สร้าง Project Stock
  2. Popup: ตั้งชื่อโครงการ + กรอกจำนวน LBS เป้าหมาย (ระบบเติมยอดรวมลูกค้าให้ แก้ได้) + แก้ Stock No./Job No./BOM No. ได้
  3. กดยืนยัน → ระบบสร้าง Stock + ย้ายไปแท็บ Stock No.

แท็บกิ่งย่อย → Stock No. (การ์ดรายโครงการ พับได้)

  • กางการ์ด → จัดการ Budget & Flow (งบ/กำไร/กระแสเงินสด) + ตรวจ LBS reconcile (ยอด LBS ใน BOM ต้องตรง เป้าหมาย Stock · เขียว=ตรง / แดง=ขาด-เกิน)
  • Material List (BOM) — สร้างได้หลายครั้ง: ปุ่ม “เพิ่ม Material List (BOM) ครั้งใหม่” สร้างรอบใหม่ ชื่อ BOM-YYYY-NNN (ครั้ง1) / (ครั้ง2) … — แต่ละครั้งมีรายการ + ปุ่ม “ส่ง BOM (ครั้งN) ให้จัดซื้อ” แยกกัน (ส่งแล้วล็อกแก้ไขเฉพาะรอบนั้น)
  • รายการ BOM: Epicor Code, Description, หมวด (LBS/Accessory), Due Date, IUM, จำนวน, สกุลเงิน (THB/USD + FX), Phase → Total คิดเป็นบาท
  • Serial LVB / Serial OM: ช่องกรอกจะปรากฏ เฉพาะเมื่อเลือกหมวด = LBS เท่านั้น (แสดงใต้ชื่อรายการในตาราง)
  • มี 📜 Audit Timeline ประวัติส่งงาน/ตีกลับ/รับงาน · ปุ่ม Workflow → เปิด Workflow board

แท็บกิ่งย่อย → เตรียมส่งมอบ Service (ออกใบเบิกติดตั้ง — สิทธิ์การเบิกอยู่ที่ฝ่ายโครงการ)

  1. กด บันทึกใบเบิก → เลือกรายการ BOM + จำนวน (ไม่เกินคงเหลือในคลังโครงการ) + แผนดำเนินการ เริ่ม–สิ้นสุด → กด 💾 บันทึกใบเบิก (ร่าง)
  2. ขั้นนี้ยังไม่ตัดสต็อก และยังไม่ส่งให้บริการ (เป็นร่าง แก้/ลบได้)
  3. เมื่อพร้อม กด 🔁 ส่งมอบ Service ที่ใบเบิกร่างนั้น → ระบบ ตัดสต็อกของโครงการ (เบิกติดตั้ง) ตามรายการ + ส่งใบเบิก+แผนเข้า Inbox ของ Service

🔗 Flow: ส่ง BOM (รายครั้ง) → ฝ่าย Purchasing เห็นเฉพาะรอบที่ส่งแล้ว · ส่งงานต่อเฟส (Workflow) project→purchasing ต้อง: งานครบ + Job No. + ส่ง BOM ครบทุกครั้ง + LBS ตรงเป้า · ใบเบิกร่างที่ยังไม่ส่ง ไม่กระทบสต็อก

🛒 Purchasing Department (ฝ่ายจัดซื้อ)

หน้าที่: รับ BOM (รอบที่ส่งแล้ว) → ออก PO → อัปเดตสถานะของเข้า แจ้งกลับฝ่ายโครงการ

หน้าหลัก (Purchasing Department) — การ์ดต่อ Stock/BOM (พับได้, เริ่มต้นซ่อน)

  1. กางการ์ด → เห็น เฉพาะรายการ BOM ของรอบที่ฝ่ายโครงการส่งมาแล้ว (มีป้าย “(ครั้งN)” กำกับ)
  2. กด ออก PO → เลือกรายการ + ระบุจำนวนต่อรายการ (แยกออกหลายใบ/บางส่วนได้) → กรอกเลขที่ PO เอง + แนบไฟล์ PDF → มูลค่ารวมคิดอัตโนมัติ
  3. อัปเดต สถานะ PO ราย BOM (Awaiting → PO Received → Delivered) — ฝ่ายโครงการเห็นเลข PO + เปิดไฟล์ PDF ได้

แท็บกิ่งย่อย → BOM Delivered completed — รวมรายการที่ส่งมอบครบ (ระบุครั้งของ BOM)

🔗 Flow: PO ผูกกับ BOM (Project Stock) · เมื่อทุกรายการ Delivered ครบ → ฝ่ายจัดซื้อ “ส่งงานต่อ” ไป Service ได้ใน Workflow · ตีกลับฝ่ายโครงการได้ (ระบุเหตุผล)

🛠️ Service Department (ฝ่ายบริการ)

หน้าที่: รับงานตามใบเบิก → จัดทีม + เวลาจริง → ส่งมอบงาน · แต่ละใบเบิกเป็น “การ์ดงาน” มี 3 ขั้นตอน กดแล้วเลื่อนขั้นถัดไปอัตโนมัติ

การ์ดงานบริการ (ต่อ 1 ใบเบิก) — ทำตามขั้น:

  1. ขั้น 1 — รับงาน (เอกสาร): ดูรายการในใบเบิก + แผนงาน → ปริ้นเอกสาร ใบเบิก / Delivery Order (D/O) / Handover Doc ไปตรวจรับสินค้าที่คลัง (WH) → กด ✓ รับงาน (ไปขั้นถัดไปอัตโนมัติ)
  2. ขั้น 2 — จัดทีม + เวลาจริง: เลือกทีมงาน 1–2 คน (สมาชิกที่ติดคิวช่วงนั้นขึ้น ⚠) + กรอก actual Start / actual End → กด ✓ บันทึกทีม+เวลา & เริ่มงาน → ระบบ ลงคิวปฏิทินให้ทีมตามวันจริงอัตโนมัติ
  3. ขั้น 3 — ส่งมอบงาน: แนบไฟล์ PDF / รูปถ่าย หลักฐานส่งมอบ → กด ✓ ส่งมอบงาน + ลงนาม DO → ปิดงานบริการ (เก็บไฟล์ + ลงนาม DO ให้โครงการ)

ทีมงาน & Scheduling Calendar: เพิ่ม/ลบสมาชิก (กรอก รหัสพนักงาน + ตำแหน่ง (พิมพ์เอง) + โทร) · ปฏิทินรายเดือน/Roster (คลิกวันที่มีงานดู Event)

🔗 Flow: การ เบิกวัสดุติดตั้ง ทำที่ฝ่ายโครงการ (ออกใบเบิก = ตัดสต็อก) — ฝ่ายบริการเป็นผู้ รับงาน → ตรวจรับ WH → ปฏิบัติงาน → ส่งมอบ · เมื่อกดส่งมอบงาน (ลงนาม DO) ครบ จึง “ส่งงานต่อ” เป็น ปิดงาน (Closed) ใน Workflow ได้ (ระบบ tick งานฝ่ายบริการให้ตามขั้นอัตโนมัติ)

📦 คลังสินค้า (Inventory)

หน้าที่: ติดตามสต็อกจริง 2 ระดับ — คลังกลาง ↔ ต่อ Project Stock · รับเข้าตาม PO + โอนข้าม Stock (สิทธิ์การเบิกติดตั้งอยู่ที่ฝ่ายโครงการ)

  • 📥 รับตาม PO: เลือก PO ที่ค้างรับ → กรอกจำนวนรับครั้งนี้ (รับบางส่วนได้) → รับครบ → อัปเดต BOM เป็น Delivered อัตโนมัติ
  • 🔁 โอนกลาง→โครงการ: ย้ายของจากคลังกลางเข้า Project Stock (สร้างคู่ เบิกออกคลังกลาง + รับเข้าโครงการ)
  • ✏️ บันทึกเอง: รับเข้าทั่วไป / เบิกออกเฉพาะ คืนผู้ขาย หรือ ชำรุด-ตัดทิ้ง (การเบิกเพื่อ “ติดตั้งงาน” ทำผ่านใบเบิกที่ฝ่ายโครงการ ไม่บันทึกซ้ำที่นี่)
  • ⚠ ระบบบล็อกการเบิก/โอนเกินยอดคงเหลือ (สต็อกห้ามติดลบ — ทั้งฝั่งจอและ trigger ใน DB)
  • ดูยอดคงเหลือ (รับเข้า/เบิกออก/คงเหลือ) + ledger เคลื่อนไหวล่าสุด · Export Excel

🔗 Flow: รับตาม PO เชื่อมกับ PO ฝ่ายจัดซื้อ (รับครบ → po_status = Delivered) · ของต้องถูก โอนเข้า Project Stock ก่อน ฝ่ายโครงการจึงออกใบเบิกตัดสต็อกได้

🔗 สรุป Flow เชื่อมโยงทั้งระบบ

ลูกค้า ──(Sales รวบรวม)──▶ Sales Requisition (SR)  [ติดตาม PO/สัญญา รายลูกค้า]
                                  │ ส่งให้โครงการ
                                  ▼
   (Project เลือกหลาย SR + ถาม LBS) ──▶ Project Stock + BOM (ได้หลายครั้ง)
              │ ส่ง BOM (รายครั้ง)                    │ ออกใบเบิก: บันทึกร่าง → ส่งมอบ
              ▼                                        ▼  (ตัดสต็อกโครงการ)
        Purchasing (ออก PO)              Service (3 ขั้น: รับงาน → จัดทีม+เวลาจริง → ส่งมอบ+แนบไฟล์)
              │ Delivered ครบ                         │ ตรวจรับ WH · ลงนาม DO
              ▼                                        ▼
   คลัง: รับตาม PO ─▶ โอนเข้า Project Stock ───────────▶  ปิดงาน (Closed)

เฟสของ Project Stock (Workflow): project → purchasing → service → closed

เงื่อนไขส่งงานต่อ (Gate) แต่ละเฟส:

จาก → ไปต้องผ่านเงื่อนไข
project → purchasingงานฝ่ายโครงการครบ + มี Job No. + มี BOM + ส่ง BOM ครบทุกครั้ง (ทุกรายการ) + LBS ใน BOM = เป้าหมาย
purchasing → serviceงานฝ่ายจัดซื้อครบ + ทุกรายการ BOM = Delivered
service → closedงานฝ่ายบริการครบ + ลงนาม DO แล้ว

ส่ง-รับ-ตอบกลับ ข้ามฝ่าย (Workflow board):

ส่งงานต่อ (Forward): เจ้าของเฟสกดส่งไปเฟสถัดไป (ผ่าน Gate เท่านั้น) → ปลายทางเห็น “งานเข้าใหม่”

รับงาน (Ack): ฝ่ายปลายทางกด “รับงาน” เพื่อเคลียร์ badge งานเข้าใหม่

ตีกลับ (Reject): เจ้าของเฟสส่งกลับเฟสก่อนหน้าได้ (ต้องระบุเหตุผล) → บันทึกใน Audit Timeline + แจ้งเตือนกระดิ่ง
