📋 Sales Department (ฝ่ายขาย)

หน้าที่: เก็บข้อมูลลูกค้า → รวบรวมเป็นใบขอ (SR) ส่งให้ฝ่ายโครงการ → ติดตาม PO/สัญญาจากลูกค้า

หน้าหลัก (Sales Department) — แสดงเฉพาะลูกค้าที่ยังไม่ได้รวบรวม

  1. กด + เพิ่มลูกค้า กรอก: ชื่อ, ประเภทลูกค้า, ประเภทงาน, ผู้ติดต่อ, จำนวน LBS ที่ต้องการ, วันที่ต้องการ, สถานที่ติดตั้ง, Scope
  2. ลูกค้าจะถูกจัดกลุ่มตามประเภท (ราชการ / เอกชน / รัฐวิสาหกิจ)
  3. ติ๊กเลือกลูกค้า (เลือกได้หลายราย) → กด รวบรวมส่งฝ่ายโครงการ → ระบบออกเลข SR-YYYY-NNN

แท็บกิ่งย่อย → Sales Requisition & ติดตาม PO

  • กล่อง ⏳ รอฝ่ายโครงการสร้าง Stock / ✓ สร้าง Project Stock แล้ว
  • กรอก เลขที่ PO จากลูกค้า + สถานะสัญญา เพื่อติดตาม

🔗 Flow: เมื่อกด "รวบรวมส่ง" → SR จะไปโผล่ใน Inbox ของ Project ทันที · ลูกค้าจะถูกย้ายออกจากหน้าหลัก Sales (ไปอยู่ใต้ SR)

📐 Project Department (ฝ่ายโครงการ)

หน้าที่: รับ SR → สร้าง Project Stock → ทำ BOM → ส่งให้จัดซื้อ → แจ้งแผนส่งมอบให้บริการ

หน้าหลัก (Project Department) — Inbox ของ SR ที่ฝ่ายขายส่งมา

  1. ติ๊กเลือก SR (เลือกได้หลายใบ) → กด สร้าง Project Stock
  2. Popup: ตั้งชื่อโครงการ + กรอกจำนวน LBS เป้าหมาย (ระบบเติมยอดรวมลูกค้าให้ แก้ได้) + แก้ Stock No./Job No./BOM No. ได้
  3. กดยืนยัน → ระบบสร้าง Stock + ย้ายไปแท็บ Stock No.

แท็บกิ่งย่อย → Stock No. (การ์ดรายโครงการ พับได้)

  • กางการ์ด → จัดการ Budget & Flow (งบ/กำไร/กระแสเงินสด) + Material List (BOM)
  • เพิ่มรายการ BOM: Epicor Code, Description, Due Date, IUM, จำนวน, สกุลเงิน (THB/USD + FX), Project Phase → Total คิดเป็นบาท
  • ตรวจสอบจำนวน LBS: ยอด LBS ใน BOM ต้องตรงกับ เป้าหมาย Stock (สีเขียว = ตรง / แดง = ขาด-เกิน)
  • กด ส่ง BOM ให้จัดซื้อ (ส่งแล้วล็อกแก้ไข) · กด ส่งแผน Service (วันส่ง/สถานที่/ทีม)
  • มี 📜 Audit Timeline ประวัติส่งงาน/ตีกลับ · ปุ่ม Workflow → เปิด Workflow board

🔗 Flow: ส่ง BOM → ไปฝ่าย Purchasing · ส่งแผน → เข้า Inbox ของ Service · ส่งงานต่อเฟส (Workflow) ต้องผ่านเงื่อนไข: มี Job No. + ส่ง BOM แล้ว + LBS ตรงเป้า

🛒 Purchasing Department (ฝ่ายจัดซื้อ)

หน้าที่: รับ BOM → ออก PO → อัปเดตสถานะของเข้า แจ้งกลับฝ่ายโครงการ

หน้าหลัก (Purchasing Department) — การ์ดต่อ BOM No. (พับได้, เริ่มต้นซ่อน)

  1. กางการ์ด → เห็นรายการ BOM ครบทุกคอลัมน์
  2. กด ออก PO → เลือกรายการ + ระบุจำนวนต่อรายการ (แยกออกหลายใบได้) → กรอกเลขที่ PO เอง + แนบไฟล์ PDF → มูลค่ารวมคิดอัตโนมัติ
  3. อัปเดต สถานะ PO ราย BOM (Awaiting → PO Received → Delivered) — ฝ่ายโครงการเห็นเลข PO + เปิดไฟล์ PDF ได้

แท็บกิ่งย่อย → BOM Delivered completed — รวมรายการที่ส่งมอบครบทุก PO

🔗 Flow: PO ผูกกับ BOM No. (Project Stock) · เมื่อทุกรายการ Delivered ครบ → ฝ่ายจัดซื้อ "ส่งงานต่อ" ไป Service ได้ใน Workflow

🛠️ Service Department (ฝ่ายบริการ)

หน้าที่: รับแผนส่งมอบ → จัดทีม/ตารางคิว → ลงนาม DO → ปิดงาน

  1. 📥 Inbox แผนส่งมอบใหม่ จากฝ่ายโครงการ → กด รับแผนงาน
  2. การ์ดโครงการพร้อมส่งมอบ: เช็กลิสต์งาน + ลงนาม DO (เสร็จแล้วปิดงานได้)
  3. ทีมงาน & Scheduling Calendar: เพิ่ม/ลบสมาชิก · ปฏิทินรายเดือน (คลิกวันที่มีงานดู Event) · มอบหมายทีมเข้าโครงการ

🔗 Flow: ต้อง ลงนาม DO ก่อน จึง "ส่งงานต่อ" เป็น ปิดงาน (Closed) ใน Workflow ได้

📦 คลังสินค้า (Inventory)

หน้าที่: ติดตามสต็อกจริง 2 ระดับ — คลังกลาง ↔ ต่อ Project Stock (บันทึกรับ/เบิกด้วยมือ)

  • 📥 รับตาม PO: เลือก PO ที่ค้างรับ → กรอกจำนวนรับครั้งนี้ (รับบางส่วนได้) → รับครบ → อัปเดต BOM เป็น Delivered อัตโนมัติ
  • ✏️ บันทึกเอง:
    • รับเข้า (เข้าคลังกลาง/โครงการ)
    • เบิกออก — มีประเภท: ติดตั้งงาน / คืนผู้ขาย / ชำรุด-ตัดทิ้ง
    • โอนเข้าโครงการ — ย้ายของจากคลังกลาง → Project Stock
  • ⚠ ระบบบล็อกการเบิก/โอนเกินยอดคงเหลือ (สต็อกห้ามติดลบ)
  • ดูยอดคงเหลือ (รับเข้า/เบิกออก/คงเหลือ) + ledger เคลื่อนไหวล่าสุด · Export Excel

🔗 Flow: รับตาม PO เชื่อมกับ PO ของฝ่ายจัดซื้อ (รับครบ → po_status = Delivered) · โอน/เบิกผูกกับ Project Stock

  2. จัดการข้อมูลทุกฝ่าย: ดูภาพรวม + ลบข้อมูลข้ามฝ่าย

🔗 สรุป Flow เชื่อมโยงทั้งระบบ

ลูกค้า ──(Sales รวบรวม)──▶ Sales Requisition (SR)
                                  │ ส่งให้โครงการ
                                  ▼
         (Project เลือกหลาย SR + ถามจำนวน LBS) ──▶ Project Stock + BOM
                                  │ ส่ง BOM                    │ ส่งแผน
                                  ▼                            ▼
                            Purchasing (ออก PO)          Service (รับแผน)
                                  │ Delivered ครบ              │ ลงนาม DO
                                  ▼                            ▼
              คลังสินค้า: รับตาม PO ─▶ โอนเข้าโครงการ ─▶ เบิกติดตั้ง      ปิดงาน (Closed)

เฟสของ Project Stock (Workflow): project → purchasing → service → closed

เงื่อนไขส่งงานต่อ (Gate) แต่ละเฟส:

จาก → ไปต้องผ่านเงื่อนไข
project → purchasingงานฝ่ายโครงการครบ + มี Job No. + มี BOM + ส่ง BOM แล้ว + LBS ใน BOM = เป้าหมาย
purchasing → serviceงานฝ่ายจัดซื้อครบ + ทุกรายการ BOM = Delivered
service → closedงานฝ่ายบริการครบ + ลงนาม DO แล้ว

ตีกลับ (Reject): เจ้าของเฟสกดส่งกลับเฟสก่อนหน้าได้ (ต้องระบุเหตุผล) → บันทึกใน Audit Timeline + แจ้งเตือนกระดิ่ง

รับงาน (Ack): เมื่อมีงานส่งต่อมา ฝ่ายปลายทางกด "รับงาน" เพื่อเคลียร์ badge งานเข้าใหม่
